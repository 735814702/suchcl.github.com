input的数字控件

html5为input增添了不少新的属性，为我们的开发带来了很大的便利，但在带来便利的同时，也带来了一些问题。

数字控件

这是HTMl5中新增的一个属性。

这样，就生成了一个只能输入数字的数字控件。既然是一个数字控件，很简单了，就是说这个输入框控件只能够输入数字，其他类型的文案，是不能正常输入的，比如汉子、英文字母、标点符号等，但在一些场景下可以输入运算符。

概念

MDN给出的概念是用于输入浮点数的控件，W3C的定义是带有spinner控件的数字字段，结果总的来说是一个意思，就是这个控件是用来输入数字的，数字可以是整型、浮点型。这个控件不接受非浮点型文本的输入，如果在数字控件中输入非数字的文本，是不能输入的。

除了可以输入数字类型值以外，还可以在该控件中非第一个位置输入运算符。虽然输入了运算符，但是并不能进行数学计算，如果这个时候取值会得到一个undefined。

默认值

这个数字控件在没有输入任何值的情况下的默认值是undefined。

行为交互

> IE8及以下浏览器，不考虑这种交互行为

从概念我们可以知道，这个控件是一个带有spinner控件的数字字段，那么就说明了这个控件有一些上下拉的行为。

默认样式

以chrome为代表的webkit内核的浏览器

数字控件在大部分的现代PC浏览器(支持HTML5、CSS3的浏览器)中对默认样式和普通的文本输入框没有不同，但是当有鼠标滑过或者获取到焦点的时候，就会在控件右侧出现向上向下的箭头，点击箭头并可以对控件值进行加1或减1的操作(如果控件值为浮点型的，会对小数部分去尾取整)。

firefox浏览器

数字控件在firefox中默认就展示了带有上下的箭头。

移动端(手机)浏览器

移动端浏览器主要是webkit内核的浏览器，默认没有显示上下箭头。

默认行为

虽然数字控件的默认样式在不同的浏览器的表现略有不同，但是默认行为缺是保持了一致。

无论webkit内核的浏览器，还是firefox浏览器，点击上下箭头都会触发change事件，并且带来控件值的加1或减1的操作，如果控件值为小数，那么会对该值进行去尾取整，然后进行加1或减1的操作。

还有一个默认行为，就是在控件获取到焦点到时候，点击键盘上的上下箭头也会触发change事件，会有值的加1或减1的操作。

控件在有加1或减1操作的时候，会有一下几种情况：

  1. 控件值是整数，那么值是正常的加1、减1；

  2. 控件值是浮点型(带有小书点)，会对值去尾取整后加1减1；

  3. 控件中值是一个能输入的但非正常数值型数据但时候，进行的加1、减1操作是以0为基准的，如空间中输入了3.4.5,点击控件右侧的上下箭头或键盘上的时候，是按照当前值为0来进行加1、减1操作的。

如果在控件内输入了一个伪数字，就是一个看起来像一个数字但又不是一个真正意义上数字的时候，如2.4.5等有多个小数点的情况，就不会触发change事件。

去除默认样式

数字控件有上下箭头，虽然在一定的场景会方便我们的行为，但我们的初衷并不是为了点击这里的上下箭头来实现我们的一些操作(这个默认的UI实在是太简陋了，不小心是点不到的，或者对不愿意使用键盘的人来说这简直就是一个糟糕透顶的交互)，而是希望控件可以帮助我们做一些相应的数值型的验证，但这方面做的并不友好。那么我们要么就不使用这个数字控件，或者去掉控件的默认样式。

总的来说，这个控件在开发中并没有为我们带来多少便利，除了一个不怎么友好的非数字不能输入(个别情况除外)的验证，对于我们复杂的业务验证，还是少不了js的支持，所以，如果不是为了赶时髦，并且有技术团队支持的话，暂时就当这个控件不存在吧。

github+hexo搭建个人网页

曾经，有过这么一个想法，能把自己的见闻记录下来，以使自己在多年以后还能留下点什么，于是就萌生了个这样的想法，搭建一个个人博客。

看到别人都自己买空间、域名，用wordpress或其他的博客程序加起来一个博客，瞬间感觉高大上了学多，但是这并不是自己想要的。

如果希望有一天，自己的博客或网站访问量大了，加几个广告位，倒是一个不错的注意，而我其实是最讨厌网页中的广告的，我们在优酷或者爱奇艺追个剧吧，一开始先来90秒的广告。所以我追求的就是干净，不要有广告的袭扰。所以，我最终选择了github，好处多多。首先，我每年可以省下几十块的域名费用，还有一部分的服务器费用，也不用自己维护服务器，最重要的是省去了备案的麻烦。

好了，开始正题。

建站工具

github + hexo

是什么hexo？

  • hexo是一个基于nodejs的、快捷、简介且高效的静态博客框架。Hexo使用markdown或其他的渲染引擎解析文章，在几秒钟内就可以利用相关主题生成的静态网页，并发布到github。

环境搭建

  • nodejs:前面已经说了，hexo是基于nodejs的博客框架，那么就首先需要安装nodejs;

  • 申请github账号：因为我是要把代码部署到github，所以要提前申请一个github的账号。

  1. 新建本地仓库，如新建blog

    到此位置，hexo的安装已经全部完成了,接下来，我们就可以搭建github环境。

HTML标记嵌套规则

  • html元素分为块状元素和内联元素，块状元素作为其他元素的容器
  • 块状元素可以嵌套其他块状元素
  • 块状元素可以嵌套内联元素
  • 内联元素不能嵌套块状元素
  • p内不能嵌套块状元素
  • 有几个特殊块状元素只能包含内联元素，而不能包含块状元素，这几个元素是:
  • h1、h2、h3、h4、h5、h6、p、dt
  • 块状元素和块状元素并列，内联元素和内联元素并列
  • li里面可以嵌套div
  • 下拉列表框option中只能嵌套文本
  • ol和li之间、ul和li之间，只能有注释节点或以空格显示的文本节点
